# Meowthm
## 一、 项目全局与核心设定
- 开发团队：软喵队 
- 基础模式：4k 定轨下落式音游 
- 游戏主体底层渲染：根据时间和流速计算图形化显示
## 二、 界面路由层 (UI 模块)
**全局采用 QStackedWidget 结合多个独立子类，实现不同界面的丝滑切换，每个画面均封装为一个单独的类 。**
- **主流程界面**：	
  - 游戏初始界面 
  - 设置界面（用于延迟设置） 
  - 选曲界面 （需要selected变量记录选曲）
  - 游戏进行界面 (4k轨道) 
  - 结算界面 
- **模态打断界面**：
  - 暂停界面：由点击 ESC 键触发，弹出一个模态的 QDialog 。提供暂停游戏、重新开始、退出游戏等辅助功能 。 
- **拓展界面 (低优先级)**：个人资料界面、成就界面、自定义曲谱编辑界面 。 
## 三、游戏数据准备
- **谱面解析器 (Map Parser)**：在游戏开始前介入。负责读取谱面文件，利用 BPM 公式将所有的“第 x 小节第 y 拍”计算转换为“距离开局的绝对毫秒数”，随后一次性将这些算好时间的 Note 对象分配塞入对应的 Track 容器中 。 （需要指定文件具体书写格式）
- **全局计时器**：使用 QElapsedTimer 实现整体的主计时 。 
- **延迟补偿变量**：设有一个偏移参数，允许调整音频与代码运行之间可能存在的延迟时间 。 
## 四、 核心游戏调度 (GameScene)
*GameScene 是主控台类，负责统筹轨道、输入与全局状态 。*
- 组件持有：内部包含 4 个 Track 轨道的实例，以及分数计数器和连击计数器 。 
- 输入监听与映射：通过无限循环持续捕捉键盘输入以实现交互 。默认按键映射为 D、F、J、K，若时间允许可实现玩家自定义功能 。 
- 事件下发机制：作为拥有资格接收系统键盘事件和全局时间的第一层级，GameScene 会在捕获按键动作时，携带着当前绝对时间，直接将具体的判定指令下发给对应轨道的 checkHit(时间参数) 函数 ，或者触发暂定界面。
## 五、 数据实体与判定层 (Track & Note)
- Track (轨道类)：作为第二层级的核心判定发生地，包含持有音符的容器。负责接收上层（GameScene)的指令，并执行具体的 checkHit 判定逻辑 。 
- Note (音符基类)：记录流速（目前统一流速）、判定时间、具体的拍子以及 BPM 。 
  -Tap (单点子类)：继承自 Note 类。触发条件为音符下落至与判定线重合时（按时间判定），玩家按下对应键盘按键 。 
  - Hold (长按子类)：继承自 Note 类 。其判定机制拆解为三步： 
    - 头判：音符头部与判定线重合时按下 。
    - 中途判定：检查玩家是否保持按住，如果中途松手则判定断连 。 
    - 尾部结算：无尾判要求，当音符尾部越过判定线时事件自动结束 。 
## 六、 视听反馈系统 (Feedback)
* 该模块用于在判定发生时，向玩家提供即时操作反馈。*
- 音频控制模块：
  - BGM (背景音乐类)：通过 QMediaPlayer 实现。负责播放长曲目，并向整个系统提供当前播放的绝对时间 。 
  - SFX (打击音效类)：通过 QSoundEffect 实现。在玩家每次敲击键盘时触发“喵”音效 。/ 或者和视觉反馈一样，在判定时调用发出声音。
- 打击视觉反馈类：当点击时间与轨道容器内判定时间发生交互后，在此类中实现对应的图形特效化呈现 。
